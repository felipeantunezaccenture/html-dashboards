General Considerations
💡
Clarity & Simplicity
Use clear, full sentences.
Avoid unnecessary complexity.
🎯
Specificity
Be precise, generic prompts lead to generic responses.

Prompt Length
e.g. 'more specific', '3 paragraphs'
Domain Info
e.g. age group, profession, interest
Avoid Acronyms
e.g. spell out 'S&OP' fully
Perspectives
e.g. 'objective', 'neutral', 'critical'
Tone of Prompt
e.g. 'casual', 'urgent', 'formal'
Audience
e.g. 'economic impact', 'productivity'
Define the Role, 'Flip the role'
e.g. 'devil's advocate', 'consulting partner', 'expert in supply chain'
Best Practices / Recommendations

A structured approach to consistently deliver executive-ready, high-quality outputs.

1
Structure Before Speed
Use Thinking mode and Projects to create consistency across outputs.

Define the Source of Truth

Target Precisely
File name + exact location + task + format in every prompt.

Run a QA pass and assumptions check before sharing externally.

What are Claude Skills?
⚙️
Reusable instructions for Claude
A Skill is a saved set of instructions stored in Claude Code that automatically shapes how Claude responds in a specific situation, like a template it always follows.
🔁
Triggered by a slash command
You invoke a skill by typing /skill-name in Claude. It then follows those instructions exactly, every time.
🎯
Consistent quality, zero re-prompting
You write the instructions once. From then on, Claude always produces outputs that follow your standards, no need to repeat yourself each session.
How to create a Skill
1
Describe the skill to Claude in the chat
Tell Claude what you want it to always do, e.g. "Create a skill called accenture-html that always follows our brand when building HTML pages."
2
Claude creates the folder and file automatically
Claude saves it automatically in your project so it's available every time you open a new session.
3
Invoke it any time with /skill-name
From that moment on, type /accenture-html at the start of any Claude session and it follows your instructions automatically.
💡
You can also create the file manually: make a .claude/commands/skill-name.md file and write the instructions inside. The filename becomes the slash command.

10 Prompts Ready to Use, Start with One

We've pulled together prompts we actually use every day. Browse them, pick the one that feels most useful right now, and give it a go, small steps make a big difference.

⚡

One prompt a day keeps the learning going

These are real prompts for real work, emails, meetings, decks, data. Just swap in your context where you see brackets and paste. That's it. Start with whichever feels most useful today.

01
Returning from PTO or needing to catch up quickly!
Summarize my emails, Teams messages and channel messages from the last workday. List action items in a dedicated column. Suggest follow-ups if possible in a dedicated column. The table should look like this: Type (Mail/Teams/Channel) | Topic | Summarization | Action item | Follow-up. If I have been directly mentioned, make the font of the topic bold.
02
Summarize a Teams meeting and provide comprehensive minutes including action items.
Please generate detailed meeting minutes. Include the following elements in your output: * **Meeting Title:** Provide the title or a concise description of the meeting's purpose. * **Date and Time:** Clearly indicate the date and time the meeting took place. * **Attendees:** List all the participants who were present at the meeting. If possible, identify their roles or affiliations. * **Agenda Items:** Organize the meeting minutes based on the agenda items discussed. Clearly identify each agenda item and summarize the key points, discussions, and decisions related to it. * **Action Items:** Identify and list any specific action items that emerged from the meeting. For each action item, specify the person responsible for completing it and the deadline, if mentioned. * **Next Steps:** If there were any discussions or decisions about the next steps to be taken after the meeting, summarize them concisely. * **Other Notes:** Include any additional information or important details that were discussed during the meeting but don't fit into the above categories. **Important Considerations:** * **Accuracy:** Ensure that your summary accurately reflects the discussions and decisions made in the transcript. * **Conciseness:** Aim to be concise and focused in your summary while capturing all the essential information. * **Clarity:** Use clear and straightforward language to present the information in an easily understandable manner. * **Objectivity:** Maintain an objective and neutral tone throughout the summary. **Additional Tips:** * If the transcript is long, consider breaking it down into smaller sections to make the summarization process more manageable. * Use bullet points and numbered lists to organize information for better readability. * Proofread your summary carefully to eliminate any errors or inconsistencies. **Please provide a detailed summary of the meeting.** **Format the summary in markdown format.**
03
Prepares for the next workday with summaries of messages, tasks, meetings, and weather.
As my personal assistant, help me prepare for my next working day (Monday to Friday). 1. Use a greeting and mention the day of the week and today's date in a large, bold font. 2. Summary of Today: Summarize unread emails from today, grouping similar messages together. Summarize Teams chat messages from today, grouping similar messages together. Include a table of open tasks assigned to me from the emails and chat messages above, with due dates. Review each task and only include open tasks assigned to me. Exclude completed tasks and tasks assigned to others. 3. If today is Friday, add a section For Next Week:, table of Next Monday's meetings (Time, Event, Organizer). Table of open tasks assigned to me with future due dates (Task, Due Date, Assigned By, Origin). Review each task and only include open tasks assigned to me. Exclude completed tasks and tasks assigned to others. Weekend and next week's weather forecast in Fahrenheit. 4. If today is not Friday, add a section For Tomorrow:, table of Tomorrow's meetings (Time, Event, Organizer). Table of open tasks assigned to me with future due dates (Task, Due Date, Assigned By, Origin). Review each task and only include open tasks assigned to me. Exclude completed tasks and tasks assigned to others. Tomorrow's weather forecast in Fahrenheit. If the temperature is between 50°F and 65°F, recommend a light jacket. If the temperature is below 50°F, recommend a winter coat. If there is a chance of rain, add a recommendation in bold to bring an umbrella when I leave the house. 5. Get Prepared: Summarize my day and brief me on all the important things I need to know to make my next working day productive.

You are a change lead preparing for a [describe project in brief] with a [describe the client org]. Create a [describe the duration] communications plan within a table format.
