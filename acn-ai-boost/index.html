Challenge 1

Doc2Pitch · Advanced Prompting

Your first warm-up: design master prompts that turn documents into a professional pitch, executable across multiple AI models.

Objective

Turn documents into a compelling pitch

Create a professional pitch from one or more documents uploaded to a GenAI tool (ChatGPT, Claude, Perplexity, ARC, etc.), designing a complex, reusable prompt that:

  • Reads, interprets and synthesizes the source documents.
  • Generates a structured pitch (text, script or slides).
  • Controls output tone, format and style.
Level 1 Advanced prompting Doc → Pitch
Functional requirements
Input One or more documents (PDF, DOCX or text). Any topic.
Process
  • Upload the documents to the chosen tool.
  • Design and test a master prompt that synthesizes and generates the pitch.
  • Run the same prompt on at least two different models and compare results.
Output
  • Final pitch (text or presentation).
  • Full prompt used.
Challenge 2

Doc2Podcast · Notebook LM

We go from text to audio: transform knowledge into a spoken, narrative format ready to share.

Objective

Create a podcast from a document

Use Notebook LM to convert the document “Intelligent Digital Brain” into a podcast structured as an interview, in English.

Level 2 Text → Audio Notebook LM
Functional requirements
Input Source document: “Intelligent Digital Brain” (View doc).
Language English.
Format Podcast in interview format.
Output
  • Generated audio file or a shareable link to the podcast.
Challenge 3

Meeting2Action · Agent + Knowledge

This is no longer about “using AI”, but about building an agent that drastically reduces the cost of preparing and sending meeting minutes.

Objective

From transcription to a ready-to-send email

Build an agent that receives a meeting transcription + your notes and generates:

  • Clean, structured, “client-ready” minutes following a predefined template.
  • A complete email (subject, recipients, body) ready to copy and paste.
Level 3 Agent + Knowledge Meeting → Minutes → Email
Functional requirements
Input
  • Recording / transcription of a meeting.
  • Your written notes (bullets, highlights, decisions, etc.).
Process
  • Define a minutes template (structure, tone, length, format).
  • The agent merges transcription + notes and generates the minutes.
  • After your review, the agent generates a complete email (Subject, To/CC, Body).
Output
  • Final minutes approved.
  • Email ready to send to client / team.
Challenge 4

Acn2Agent · News + APIs + KX

We take a leap in autonomy: design an agent that understands Accenture's external and internal context, filters noise and delivers actionable insight.

Objective

News agent for Accenture-relevant updates

Design an agent capable of detecting the most recent news about Accenture, combining:

  • External sources (public APIs and/or web scraping).
  • Internal sources (KX or other authorized sources).

The agent should filter, rank and explain why each piece of news matters for Accenture.

Level 4 Agent + APIs News Intelligence
Functional requirements
Input
  • External searches via APIs or scraping.
  • Queries to KX or other authorized internal sources.
Process
  • Find relevant news about Accenture.
  • Consult internal sources to enrich context.
  • Apply impact/relevance filters (trend, reach, market, affected service, etc.).
  • Rank news by potential impact.
Output
  • Structured summary with:
    • Brief description of each piece of news.
    • Why it matters.
    • Potential implications for Accenture.
    • Link to the original source.
Challenge 5

Acn2Web · Replicate HTML

We start playing seriously with HTML: reproduce a newspaper front page and tell how Accenture is revolutionizing the market with AI.

Objective

Newspaper landing + Accenture & AI story

Replicate the landing page of any newspaper (national or international) in HTML using a tool like v0 or Replit, including:

  • Newspaper front-page structure (header, lead story, secondary sections, sidebar).
  • A feature story about how Accenture is revolutionizing the market with AI.
Level 5 Replicate HTML Storytelling · AI
Functional requirements
Input
  • Reference URL of the real front page (for visual inspiration only).
  • Fictional content for the main story (headline, text and image/placeholder).
Process
  • Create a static HTML/CSS page replicating the newspaper structure.
  • Include a feature story about Accenture + AI.
  • Implement responsive design and basic accessibility.
Output Functional HTML page that reproduces the look & feel of a newspaper front page with the Accenture & AI story.
Challenge 6

Data2Report · HTML Reporting

From replication to creation: design a professional, interactive HTML report ready for a client.

Objective

Interactive, client-ready HTML report

Design and generate, using AI, a professional, interactive HTML report that visualizes data and insights at a quality suitable to share with a client or leadership.

Level 6 HTML + Visualization Data2Report
Functional requirements
Input
  • Dataset or data source (Excel, CSV, internal data…).
  • Clear business context (e.g., project status, market analysis, performance dashboard…).
Process
  • Define report structure: executive summary, key metrics, visualizations, conclusions, recommendations.
  • Use AI to generate the complete HTML/CSS code.
  • Include at minimum:
    • Header with title and date.
    • 2–3 visualizations (charts, tables, visual elements).
    • Summary with key insights.
  • Ensure responsive design and add interactive elements (hover, collapsible sections, basic filters…).
Output Functional HTML file ready to open in any browser and share with client / leadership.
Challenge 7

Forms2RPing · Power Automate

We enter precision territory: automate form follow-ups and reminders in Teams without manual intervention.

Objective

Automate Forms reminders in Teams

Design and implement an automation in Power Automate that:

  • Monitors responses to a Microsoft Form.
  • Compares responses against a target recipient list.
  • Sends personalized Teams reminders to anyone who hasn't responded.
Level 7 Automation Forms → Teams
Functional requirements
Input
  • Microsoft Form.
  • Recipient list (Excel/SharePoint) with name and email/UPN.
Process
  • Create a flow with the trigger “When a new file is added to the folder” to record that a file was submitted via the form.
  • Create a scheduled flow (daily or hourly) that reads:
    • The full recipient list.
    • The stored responses.
  • Use conditions / Apply to each / Filter array to identify non-responders.
  • Send a Teams message with a link to the Form to each non-responder.
Output Flow in Power Automate that:
  • Detects non-responders.
  • Sends personalized automatic reminders.
  • Updates a small log/status (completion percentage, pending, etc.).
Challenge 8

Flow2System · Full orchestration

The final level: design an end-to-end system that combines trigger, input, transformation and output, with or without human-in-the-loop.

Objective

Design an end-to-end agentized system

Build a workflow that:

  • Ingests information.
  • Transforms it using AI or logic.
  • Produces a deliverable with business value.

You must choose at least one “topping” per block (Trigger, Input, Transformation, Output) and optionally add a Human in the loop.

Level 8 Orchestrated System Flow2System
System blocks
Block 1 · Trigger Manual / Scheduled / On file change / On email/Teams / Webhook (choose ≥1).
Block 2 · Input Local/Cloud files (PDF, Excel, DOCX) or APIs / Web scraping.
Block 3 · Transform Simple code, single agent call or Agent ecosystem (orchestrator + specialists).
Block 4 · Output Update files, send emails/Teams or generate an HTML page.
Block 5 · Human in the loop Optional: interactive HTML + webhook for approvals, editing, re-run, etc.

Quick FAQs

Frequently Asked Questions

  • Do I have to complete all challenges?
    Yes — the curriculum is progressive. Each level builds on the previous: prompting → agents → automation. Missing a submission counts as an incomplete exercise.
  • Can I work in a team?
    No — the goal is for everyone to use the tools and have an active role (prompt design, agent architecture, workflow construction, etc.).
  • How will submissions be evaluated?
    Evaluation will mainly consider:
    • Clarity of design (prompt, agent or flow).
    • Degree of automation / agentization achieved.
    • Quality of the deliverable (client-ready / senior-internal ready).
  • Who can I contact if I have questions?
    You will receive access to a Form with Q&As and weekly replies via Workflows.
